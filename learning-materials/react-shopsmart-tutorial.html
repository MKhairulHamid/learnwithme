What is React?

React is a JavaScript library for building user interfaces, especially single-page applications. It was developed by Facebook and is now one of the most popular frontend frameworks.

Why React?

  • Component-Based: Build encapsulated components that manage their own state
  • Declarative: Makes your code more predictable and easier to debug
  • Learn Once, Write Anywhere: Can be used for web, mobile (React Native), and desktop apps
  • Large Ecosystem: Tons of libraries, tools, and community support

Key Concepts Overview

Components

Reusable building blocks of your UI. Like LEGO pieces that you combine to build your application.

State

Data that can change over time. When state changes, React re-renders the component.

Props

Properties passed from parent to child components. Think of them as function arguments.

Hooks

Special functions that let you use state and other React features in functional components.

JSX - JavaScript XML

JSX is a syntax extension for JavaScript. It looks like HTML but has the power of JavaScript. React uses JSX to describe what the UI should look like.

JSX Examples 
// ✅ JSX allows you to embed JavaScript expressions
const price = 29.99
const element = <h2>Price: ${price.toFixed(2)}</h2>

// ✅ You can use JavaScript logic
const isInStock = true
const statusElement = <p>{isInStock ? 'In Stock' : 'Out of Stock'}</p>

// ✅ You can map arrays to create lists
const products = ['Laptop', 'Phone', 'Tablet']
const listElement = (
  <ul>
    {products.map((product, index) => (
      <li key={index}>{product}</li>
    ))}
  </ul>
)

// ✅ JSX attributes use camelCase
const styledDiv = (
  <div 
    className="container"
    onClick={handleClick}
    style={{ backgroundColor: 'blue', fontSize: '16px' }}
  >
    Click me
  </div>
)

JSX Rules

  • Use className instead of class
  • Use htmlFor instead of for
  • All tags must be closed: <img /> or <br />
  • Wrap JavaScript expressions in curly braces {}
  • Return one parent element (or use Fragment <></>)

State is data that can change over time. When state changes, React automatically re-renders the component to reflect those changes. This is what makes React "reactive"!

Understanding State

  • Multiple states: You can have many useState hooks in one component

useEffect - Side Effects

useEffect runs code after the component renders. Perfect for fetching data, setting up subscriptions, or manually changing the DOM.

useEffect Syntax 
import React, { useState, useEffect } from 'react'

const DataFetcher: React.FC = () => {
  const [data, setData] = useState(null)
  const [loading, setLoading] = useState(true)

  // useEffect(function, dependencies)
  useEffect(() => {
    // This runs after component mounts
    fetchData()
      .then(result => setData(result))
      .finally(() => setLoading(false))
    
    // Optional cleanup function (runs before unmounting)
    return () => {
      // Cleanup code here
    }
  }, [])  // Empty array = run once on mount

  if (loading) return <p>Loading...</p>
  return <p>Data: {data}</p>
}

Real Example from ShopSmart CartContext

src/context/CartContext.tsx 
export const CartProvider: React.FC<{ children: React.ReactNode }> = ({ children }) => {
  const [state, dispatch] = useReducer(cartReducer, initialState)

  // Load cart from localStorage when component mounts
  useEffect(() => {
    try {
      const savedCart = localStorage.getItem(CART_STORAGE_KEY)
      if (savedCart) {
        const parsedCart = JSON.parse(savedCart)
        dispatch({ type: 'LOAD_CART', payload: parsedCart })
      }
    } catch (error) {
      console.error('Error loading cart:', error)
    }
  }, [])  // Run once on mount

  // Save cart to localStorage whenever it changes
  useEffect(() => {
    try {
      localStorage.setItem(CART_STORAGE_KEY, JSON.stringify(state.items))
    } catch (error) {
      console.error('Error saving cart:', error)
    }
  }, [state.items])  // Run whenever state.items changes

  return <CartContext.Provider value={contextValue}>{children}</CartContext.Provider>
}

useEffect Dependencies

  • useEffect(() => {}, []) - Runs once on mount
  • useEffect(() => {}, [state]) - Runs when state changes
  • useEffect(() => {}) - Runs after every render (usually avoid this!)

Creating Context - CartContext Example

src/context/CartContext.tsx (simplified) 
import React, { createContext, useContext, useReducer } from 'react'

// 1. Define the shape of your context
interface CartContextType {
  items: CartItemWithProduct[]
  totalItems: number
  totalAmount: number
  addItem: (product: Product, quantity?: number) => void
  removeItem: (productId: number) => void
  clearCart: () => void
}

// 2. Create the context
const CartContext = createContext<CartContextType | undefined>(undefined)

// 3. Create the provider component
export const CartProvider: React.FC<{ children: React.ReactNode }> = ({ children }) => {
  const [state, dispatch] = useReducer(cartReducer, initialState)

  const addItem = (product: Product, quantity: number = 1) => {
    dispatch({ type: 'ADD_ITEM', payload: { product, quantity } })
  }

  const removeItem = (productId: number) => {
    dispatch({ type: 'REMOVE_ITEM', payload: { productId } })
  }

  const clearCart = () => {
    dispatch({ type: 'CLEAR_CART' })
  }

  const contextValue: CartContextType = {
    ...state,
    addItem,
    removeItem,
    clearCart,
  }

  return <CartContext.Provider value={contextValue}>{children}</CartContext.Provider>
}

// 4. Create custom hook for easy access
export const useCart = (): CartContextType => {
  const context = useContext(CartContext)
  if (context === undefined) {
    throw new Error('useCart must be used within a CartProvider')
  }
  return context
}

API Integration - Fetching Data

Most React apps need to fetch data from a backend API. ShopSmart uses Axios for HTTP requests and React Query for data fetching and caching.

API Service Layer

src/services/api.ts 
import axios from 'axios'
import { Product, Category, Order } from '../types'

// Create axios instance
const api = axios.create({
  baseURL: '/api',
  headers: {
    'Content-Type': 'application/json',
  },
})

// Product API methods
export const productApi = {
  // GET /api/product
  getProducts: async (filters?: ProductFilters): Promise<Product[]> => {
    const response = await api.get<Product[]>('/product', { params: filters })
    return response.data
  },

  // GET /api/product/{id}
  getProduct: async (id: number): Promise<Product> => {
    const response = await api.get<Product>(`/product/${id}`)
    return response.data
  },

  // POST /api/product
  createProduct: async (product: Omit<Product, 'id'>): Promise<Product> => {
    const response = await api.post<Product>('/product', product)
    return response.data
  },

  // PUT /api/product/{id}
  updateProduct: async (id: number, product: Partial<Product>): Promise<Product> => {
    const response = await api.put<Product>(`/product/${id}`, product)
    return response.data
  },

  // DELETE /api/product/{id}
  deleteProduct: async (id: number): Promise<void> => {
    await api.delete(`/product/${id}`)
  },
}

// Category API methods
export const categoryApi = {
  getCategories: async (): Promise<Category[]> => {
    const response = await api.get<Category[]>('/category')
    return response.data
  },
  // ... other methods
}

Using React Query for Data Fetching

src/pages/HomePage.tsx 
import { useQuery } from 'react-query'
import { productApi, categoryApi } from '../services/api'

const HomePage: React.FC = () => {
  // Fetch featured products
  const { data: featuredProducts = [], isLoading: productsLoading } = useQuery(
    'featured-products',           // Query key (for caching)
    () => productApi.getProducts({ inStockOnly: true }),  // Query function
    {
      select: (data) => data.slice(0, 6),  // Transform data
    }
  )

  // Fetch categories
  const { data: categories = [], isLoading: categoriesLoading } = useQuery(
    'categories',
    categoryApi.getCategories
  )

  if (productsLoading || categoriesLoading) {
    return <CircularProgress />
  }

  return (
    <Box>
      {/* Display featured products */}
      <Grid container spacing={3}>
        {featuredProducts.map((product) => (
          <Grid item xs={12} sm={6} md={4} key={product.id}>
            <ProductCard product={product} />
          </Grid>
        ))}
      </Grid>

      {/* Display categories */}
      {categories.map((category) => (
        <CategoryCard key={category.id} category={category} />
      ))}
    </Box>
  )
}

React Router - Navigation

React Router enables navigation between different pages in your single-page application (SPA). It changes the URL and displays different components without a full page reload.

Performance Tips

Optimize Your React App

  • Use keys in lists: Help React identify which items changed
  • Avoid inline functions: Define functions outside JSX when possible
  • Lazy loading: Load components only when needed
  • Memoization: Use React.memo, useMemo, useCallback for expensive operations
  • Code splitting: Split your app into smaller chunks
