Component-Based Architecture

Building React apps as a tree of reusable components

🧩 What is Component-Based Architecture?

React applications are built by composing small, reusable pieces called components. Each component represents a part of the user interface and can be combined to create complex applications.

Think of it like LEGO blocks - you can build anything by combining simple, reusable pieces!

🔄 Reusable

Write once, use everywhere. A Button component can be used throughout your app.

🏗️ Modular

Break complex UIs into smaller, manageable pieces that are easier to understand.

🔧 Maintainable

Changes to one component don't affect others. Fix bugs in isolation.

Virtual DOM

React's secret weapon for fast, efficient updates

⚡ What is the Virtual DOM?

The Virtual DOM is a JavaScript representation of the real DOM kept in memory. React uses it to calculate the most efficient way to update the user interface.

Real-world analogy: It's like planning your room reorganization on paper before actually moving furniture around!

🧠 In Memory

Virtual DOM lives in JavaScript memory, making it lightning fast to work with.

🔍 Smart Diffing

React compares old and new Virtual DOM to find minimal changes needed.

🎯 Efficient Updates

Only the parts that actually changed get updated in the real DOM.

🛒 Virtual DOM in Ecommerce

Scenario: User adds item to cart

🎯 Watch this: When you add items to cart, React only updates the cart counter and total - not the entire page!

// Before: Cart shows 2 items, $50 total // User clicks "Add to Cart" on $20 item // After: Cart shows 3 items, $70 total // React only updates: <span className="cart-count">3</span> {/* Was: 2 */} <span className="cart-total">$70</span> {/* Was: $50 */} // Everything else stays the same! 🚀

JSX (JavaScript XML)

Writing HTML-like syntax in JavaScript

📝 What is JSX?

JSX is a syntax extension that lets you write HTML-like code inside JavaScript. It makes React components more readable and expressive.

Key insight: JSX gets compiled to regular JavaScript function calls!

📖 Readable

Looks like HTML, so it's familiar and easy to understand what the UI will look like.

💪 Powerful

You can embed JavaScript expressions using curly braces { }.

🔄 Compiled

JSX gets transformed into React.createElement() calls behind the scenes.

🛒 JSX in Ecommerce Components

// ✅ With JSX (Clean and readable) const ProductCard = ({ product }) => ( <div className="product-card"> <img src={product.image} alt={product.name} /> <h3>{product.name}</h3> <p className="price">${product.price}</p> <button className="add-to-cart"> Add to Cart </button> </div> ); // ❌ Without JSX (Hard to read) const ProductCard = ({ product }) => React.createElement('div', { className: 'product-card' }, React.createElement('img', { src: product.image, alt: product.name }), React.createElement('h3', null, product.name), React.createElement('p', { className: 'price' }, '$', product.price), React.createElement('button', { className: 'add-to-cart' }, 'Add to Cart') );
JSX Magic

Notice how we use {product.name} and {product.price} to inject JavaScript values into our HTML-like structure!

State Management

Managing data that changes over time

🗃️ What is State?

State is data that can change over time. When state changes, React automatically re-renders the component to reflect the new data.

Think of it as: Your component's memory that updates the UI when it changes!

📱 Local State

Data that belongs to one component (like form inputs, toggle states).

🌐 Global State

Data shared across multiple components (like user info, shopping cart).

🔒 Immutable

Never modify state directly - always create new objects/arrays.

🛒 Shopping Cart State Example

Props & Data Flow

How components communicate with each other

📤 What are Props?

Props (properties) are how parent components pass data to child components. They flow in one direction: from parent to child.

Think of props as: Function parameters for your components!

⬇️ One-Way Flow

Data flows from parent to child components, making apps predictable.

📋 Read-Only

Child components can't modify props - they're immutable.

🔄 Dynamic

When props change, the component automatically re-renders.

🛒 Product Grid Passing Props

// Parent component: ProductGrid const ProductGrid = () => { const products = [ { id: 1, name: "iPhone 15", price: 999, image: "iphone.jpg" }, { id: 2, name: "MacBook Pro", price: 2499, image: "macbook.jpg" } ]; const handleAddToCart = (product) => { console.log('Adding to cart:', product.name); }; return ( <div className="product-grid"> {products.map(product => ( <ProductCard key={product.id} // Special prop for React name={product.name} // String prop price={product.price} // Number prop image={product.image} // String prop onAddToCart={handleAddToCart} // Function prop /> ))} </div> ); }; // Child component: ProductCard const ProductCard = ({ name, price, image, onAddToCart }) => ( <div className="product-card"> <img src={image} alt={name} /> <h3>{name}</h3> <p>${price}</p> <button onClick={() => onAddToCart({ name, price })}> Add to Cart </button> </div> );
